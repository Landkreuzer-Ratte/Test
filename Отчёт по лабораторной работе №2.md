### 1.1 Разработка дополнительного элемента `my_func`

```vhdl
LIBRARY ieee;
USE ieee.std_logic_1164.ALL;

ENTITY my_func IS
PORT (
    D: IN std_logic;
    CLK: IN std_logic;
    Q: OUT std_logic
);
END my_func;

ARCHITECTURE behav OF my_func IS
    SIGNAL Q_OUT : std_logic := '0';
    SIGNAL should_output_one : std_logic := '0';
BEGIN
    PROCESS(CLK)
    BEGIN
        IF (CLK'event AND CLK = '1') THEN
            IF (should_output_one = '1') THEN
                Q_OUT <= '0';
                should_output_one <= '0';
            ELSIF (D = '0' AND Q_OUT = '1') THEN
                should_output_one <= '1';
				Q_OUT <= '1';
            ELSIF (D = '1') THEN
                Q_OUT <= '1';
            END IF;
        END IF;
    END PROCESS;

    Q <= Q_OUT;
END behav;
```

Данный элемент реализует расширитель импульса на один такт

На каждом переднем фронте тактового сигнала:
1. Если `D = '1'`, то `Q` становится `'1'`
2. Если `D = '0'` и `'Q' = 0`, то `Q` остаётся `1` ещё на один такт (устанавливается флаг `should_output_one`)
3. Если флаг установлен, то `Q` становится `'0'`, флаг сбрасывается

В схеме FIFO-буфера данный элемент используется для обеспечения задержки сигнала `rden` на один такт, чтобы память успела прочитать данные по новому адресу, а не по старому.

`my_func` также выполняет роль защиты от чтения из пустого буфера

### 1.4 Принцип работы FIFO-буфера

FIFO - это буфер, в котором данные записываются и считываются в порядке очереди. Первый записанный элемент будет первым считанным. Это похоже на очередь в магазине: кто первый пришел, тот первый обслуживается.

- **Входы**:
	- `DATA` - Вход для данных, которые нужно записать в буфер
	- `WR` - Сигнал записи, который активирует запись данных в буфер
	- `RD` - Сигнал чтения, который активирует чтение данных из буфера
	- `CLK` - Синхросигнал, который управляет операциями записи и чтения
- **Выходы**:
	- `OUT_DATA` - Выход для данных, которые были считаны из буфера
	- `IS_FULL` - Сигнал, указывающий, что буфер заполнен
	- `IS_EMPTY` -  Сигнал, указывающий, что буфер пуст

**Алгоритм работы FIFO-буфера**
1. Данные записываются в буфер по сигналу `WR`. Счетчик адресов записанных данных `CTR1` увеличивается
2. Данные считываются из буфера по сигналу `RD`. Счетчик адресов прочитанных данных `CTR2` увеличивается
3. Если `CTR1` и `CTR2` равны, это означает, что буфер пуст или полон, в зависимости от операции (чтения или записи)

### 2.1 Описание FIFO-буфера на языке VHDL

```vhdl
LIBRARY ieee;
USE ieee.std_logic_1164.all;

ENTITY fifo IS
    PORT(
        DATA: IN std_logic_vector(7 downto 0);
        CLK: IN std_logic;
        WR: IN std_logic;
        RD: IN std_logic;
        RESET: IN std_logic;
        OUT_DATA: OUT std_logic_vector(7 downto 0);
        IS_FULL: OUT std_logic;
        IS_EMPTY: OUT std_logic
    );
END fifo;

ARCHITECTURE behaviour OF fifo IS
    TYPE WORDS IS ARRAY (0 TO 15) OF std_logic_vector(7 downto 0);
    SIGNAL fifo_data : WORDS;                     
    SIGNAL wr_ptr : INTEGER RANGE 0 TO 15 := 0;               
    SIGNAL rd_ptr : INTEGER RANGE 0 TO 15 := 0;               
    SIGNAL count : INTEGER RANGE 0 TO 16 := 0;                 
BEGIN
    PROCESS (CLK, RESET)
    BEGIN
        IF (RESET = '1') THEN
            wr_ptr <= 0;
            rd_ptr <= 0;
            count <= 0;
            IS_FULL <= '0';
            IS_EMPTY <= '1';
        ELSIF (CLK'event and CLK = '1') THEN
            IF WR = '1' AND count < 16 THEN
                fifo_data(wr_ptr) <= DATA;
                wr_ptr <= (wr_ptr + 1) MOD 16;
                count <= count + 1;
            END IF;

            IF RD = '1' AND count > 0 THEN
                OUT_DATA <= fifo_data(rd_ptr);
                rd_ptr <= (rd_ptr + 1) MOD 16;  
                count <= count - 1;
            END IF;

            IF count = 16 THEN
                IS_FULL <= '1';
            ELSE
                IS_FULL <= '0';
            END IF;

            IF count = 0 THEN
                IS_EMPTY <= '1';
            ELSE
                IS_EMPTY <= '0';
            END IF;
        END IF;
    END PROCESS;
END behaviour;
```

### 3.1 Разработка дополнительного элемента `pause`

![[pause_scheme.png]]

Данный элемент нужен для задержки сигнала. Он построен на базе `LCELL` и `NOT` элементов.

Элемент `pause` задерживает приход тактового импульса к памяти на несколько наносекунд

- Сначала срабатывает счётчик и логика адресации. Сигнал на входах памяти успевает установится в стабильное значение
- После задержки `pause` тактовый импульс доходит до памяти
- Память видит уже стабильный адрес и корректно выполняет запись/чтение

Если память получит тактовый импульс в тот же самый момент что и счётчик, то она может попытаться считать адрес, пока тот ещё переключается. В этот момент на шине могут возникнуть ложные значения и память запишет данные по ложному адресу

### 3.4 Принцип работы LIFO-буфера

LIFO — это буфер, в котором последний записанный элемент будет первым считанным. Это похоже на стопку тарелок: последнюю положенную тарелку вы возьмете первой.

**Устройство LIFO-буфера**
LIFO строится аналогично [[Языки моделирования и описания цифровой аппаратуры/Лабораторная работа №2/Отчёт по лабораторной работе №2#1. Разработка схемы FIFO-буфера|FIFO]], но порядок чтения данных обратный. В LIFO данные записываются и считываются с одного конца (например, с вершины стека).

**Алгоритм работы LIFO-буфера**
1. Данные записываются в буфер по сигналу `WR`
2. Данные считываются из буфера по сигналу `RD`, но считывается последний записанный элемент
3. Управление адресами и сигналами `Full` и `Empty` аналогично [[Языки моделирования и описания цифровой аппаратуры/Лабораторная работа №2/Отчёт по лабораторной работе №2#1. Разработка схемы FIFO-буфера|FIFO]]

### 4.1 Описание LIFO-буфера на языке VHDL

```vhdl
library ieee;
use ieee.std_logic_1164.all;

ENTITY lifo_bhv IS
	port (
		 DATA_IN : in std_logic_vector(7 downto 0);
		 WR, RD, Reset: in std_logic;
		 CLK: in std_logic;
   	     DATA_OUT : out std_logic_vector(7 downto 0);
		 Full, Empty : out std_logic;
		 COUNT : out integer
	);
END lifo_bhv;

ARCHITECTURE behav of lifo_bhv is
TYPE numArray is array (0 to 7) of std_logic_vector(7 downto 0);
SIGNAL outElement : std_logic_vector(7 downto 0);
SIGNAL lifo : numArray;
SIGNAL isEmpty, isFull: std_logic;
SIGNAL scount : integer := 0;
BEGIN
	PROCESS(CLK, Reset)
    	variable counter : integer range 0 to 8 := 0;
	BEGIN
    	IF (Reset = '1') THEN
      		counter := 0;
			scount <= counter;
			isFull <= '0';
	     	isEmpty <= '1';
    	ELSIF (CLK'event and CLK = '1') THEN
	    		IF (WR = '1' and counter < 8) THEN 			
	        		lifo(counter) <= DATA_IN;
	        		counter := counter + 1;
					scount <= counter;
	      		END IF;
	
	   			IF(RD = '1' and counter > 0) THEN 
	   				counter := counter - 1;
					scount <= counter;
	       			outElement <= lifo(counter); 
	     		END IF;
				
	     		IF counter = 8 THEN
	     			isFull <= '1';
	     			isEmpty <= '0';
	     		ELSIF counter = 0 THEN
	     			isFull <= '0';
	     			isEmpty <= '1';
	     		ELSE
	     			isFull <= '0';
	     			isEmpty <= '0';
	     		END IF;
    	END IF;
    END PROCESS;
    DATA_OUT <= outElement;
    Full <= isFull;
    Empty <= isEmpty;
	COUNT <= scount;
 END behav;
```

### 5.1 Разработка и описание дополнительных элементов

#### 5.1.1 `cacheline`

![[cacheline_scheme.png]]

`Cacheline` - это блок кэш-памяти, который хранит данные и метки (`Tag`), указывающие, какой участок основной памяти они представляют

- **Входы**:
	- `addr[7..0]` - адрес данных в основной памяти
	- `data[7..0]` - данные, которые хранятся в кэше
	- `reset` - сигнал сброса
	- `write` - сигнал записи данных в кэш
	- `clk` - синхросигнал
- **Выходы**:
	- `out[7..0]` — данные, которые считываются из кэша
	- `hit` — сигнал, указывающий, что данные найдены в кэше (попадание)

#### 5.1.2 `BUSMUX`

`BUSMUX` - это шинный мультиплексор 2 в 1. Он выбирает одну из двух шин и передаёт её данные на выход.

- **Входы**:
	- `dataa[]` - первый входной массив данных (шина)
	- `datab[]` - второй входной массив данных (шина)
	- `sel` (select) - управляющий сигнал выбора:
		- При `sel = 0` на выход передается `dataa[]`
		- При `sel = 1` на выход передаётся `datab[]`
- **Выходы**:
	- `result[]` - выбранные данные

В схеме [[Языки моделирования и описания цифровой аппаратуры/Лабораторная работа №2/Отчёт по лабораторной работе №2#5.1.1 `cacheline`|cacheline]] он нужен для обнуления выходного значения независимо от сохранения данных

#### 5.1.3 `LPM_DFF`

`LPM_DFF` - это параметризуемый D-триггер, который используется для временного хранения данных

- **Входы**:
	- `data[7..0]` - 8-разрядный вход данных
	- `clock` - тактовый сигнал (синхронизация)
	- `enable` - сигнал разрешения (когда активен, данные записываются)
- **Выходы**:
	- `q[7..0]` - 8-разрядный выход (сохранённое значение)

**Алгоритм работы**:
1. По фронту тактового сигнала (при активном `enable`) входные данные `data[7..0]` сохраняются и появляются на выходе `q[7..0]`
2. При неактивном `enable` выходное значение остаётся неизменным

#### 5.1.4 `LPM_DECODE`

`LPM_DECODE` - это блок декодирования адресов, который определяет, какой блок кэш-памяти должен быть использован для данного адреса.

- **Входы**:
	- `data[1..0]` - данные для декодирования
	- `enable` - сигнал активации декодера
- **Выходы**:
	- `eq0`, `eq1`, `eq2`, `eq3` - сигналы, указывающие, какой блок кэш-памяти выбран

#### 5.1.5 `LPM_COUNTER`

`LPM_COUNTER` - это счетчик, который управляет адресацией и выбором блоков кэш-памяти

- **Входы**:
	- `clock` - синхросигнал
- **Выходы**:
	- `q[1..0]` - сигнал отвечающий в какой блок записать память

#### 5.1.6 `LPM_RAM_DQ`

`LPM_RAM_DQ` - это параметризуемый блок оперативной памяти (RAM)

- **Входы**:
	- `data[7..0]` - шина входных данных (8 бит). Сюда подается информация, которую нужно записать в память
	- `wren` - сигнал разрешения записи. Если он активен, данные с `data[7..0]` записываются по адресу `address[7..0]`
	- `adress[7..0]` - шина адреса (8 бит). Определяет ячейку памяти (всего 2⁸ = 256 ячеек)
	- `clock` - тактовый сигнал. Операции чтения/записи синхронизированы по фронту этого сигнала
- **Выходы**:
	- `q[7..0]` - шина выходных данных (8 бит). Отсюда считывается информация из ячейки по адресу `address[7..0]`

### 5.4 Принцип работы Кеш-памяти

Кеш-память состоит из нескольких блоков [[Языки моделирования и описания цифровой аппаратуры/Лабораторная работа №2/Отчёт по лабораторной работе №2#5.1.1 `cacheline`|cacheline]], работающих параллельно. Каждый блок кеш-памяти имеет свои входы и выходы. Блоки [[Языки моделирования и описания цифровой аппаратуры/Лабораторная работа №2/Отчёт по лабораторной работе №2#5.1.4 `LPM_DECODE`|lpm_decode]] и [[Языки моделирования и описания цифровой аппаратуры/Лабораторная работа №2/Отчёт по лабораторной работе №2#5.1.5 `LPM_COUNTER`|lpm_counter]] управляют выбором нужного блока памяти и адресацией, обходя все блоки по кругу.

**Алгоритм работы Кеш-памяти**:
1. Процессор отправляет адрес данных, которые ему нужны
2. Кэш-память проверяет, есть ли данные по этому адресу в кэше (сигнал `hit`)
3. Если данные есть в кэше (`hit = 1`), они передаются процессору
4. Если данных нет в кэше (`hit = 0`), данные считываются из основной [[Языки моделирования и описания цифровой аппаратуры/Лабораторная работа №2/Отчёт по лабораторной работе №2#5.1.6 `LPM_RAM_DQ`|памяти (RAM)]] и одновременно записываются в кэш для будущих обращений
